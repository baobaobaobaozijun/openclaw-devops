## 工作流程

```
接收任务 → 制定测试计划 → 编写测试用例 → 搭建测试环境 
→ 执行测试 → 提交 Bug 报告 → 验证修复 → 生成测试报告
```

### 详细流程

**1. 接收任务**
- 读取收件箱：`/workspace/communication/inbox/suancai/`

**2. 制定测试计划**

**3. 执行测试**
- 运行自动化测试脚本
- 记录测试结果
- 提交 Bug 报告
- 跟踪修复进度

**4. 生成报告**
- 整理测试数据
- 编写测试报告
- 提供质量评估
- 发送相关方

## Docker 环境特别说明

### 容器配置

**资源限制:**
- CPU: 0.5-1 核心
- 内存：64-128MB (最低)
- 存储：根据测试需要

**挂载卷:**
```yaml
volumes:
  - ./logs:/workspace/logs
  - ./tasks:/workspace/tasks
  - ./tests:/workspace/tests
  - ./reports:/workspace/reports
  - ./communication:/workspace/communication
```

### 多容器测试

**场景:** 需要同时测试多个服务

**方案:**
```yaml
# docker-compose.test.yml
version: '3'
services:
  suancai:
    build: .
    depends_on:
      - test-db
      - test-redis
  
  test-db:
    image: postgres:13
    environment:
      POSTGRES_PASSWORD: test
  
  test-redis:
    image: redis:alpine
```

### 容器生命周期

**启动:**
```bash
docker-compose up -d suancai
```

**查看日志:**
```bash
docker-compose logs -f suancai
```

**停止:**
```bash
docker-compose stop suancai
```
